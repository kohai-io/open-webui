## Implementation Details

### Conditional Node Features
- ‚úÖ 6 comparison operators: equals, not_equals, contains, greater, less, regex
- ‚úÖ Two output handles: "true" and "false" branches
- ‚úÖ Variable interpolation: `{{input}}` and `{{node.output.path}}`
- ‚úÖ Dynamic routing based on condition result

### Loop Node Features
- ‚úÖ 3 loop types:
  - **Count**: Simple iteration N times
  - **Array**: Iterate over array elements with path navigation (e.g., `{{websearch.output.results}}`)
  - **Until**: Loop until condition met (max iterations limit)
- ‚úÖ Two output handles:
  - **EACH**: Executes per iteration (passes `{iteration, value}`)
  - **DONE**: Executes once after all iterations (passes aggregated results)
- ‚úÖ Nested downstream execution for EACH branches
- ‚úÖ Accumulation of iteration results in output nodes

### Merge Node Features
- ‚úÖ 4 merge strategies:
  - **Concat**: String concatenation with configurable separator
  - **Array**: Combine inputs as array
  - **First**: Take first input only
  - **Last**: Take last input only
- ‚úÖ Configurable separator (supports `\n` for newlines)
- ‚úÖ Multiple input support

### Knowledge Node Features
- ‚úÖ Knowledge base selection (dropdown from available bases)
- ‚úÖ Query with variable interpolation
- ‚úÖ Top-K results (1-20 slider)
- ‚úÖ Confidence threshold (0-1)
- ‚úÖ Reranking option for better relevance
- ‚úÖ Hybrid search (semantic + keyword)
- ‚úÖ Metadata inclusion toggle
- ‚úÖ Returns structured chunks with relevance scores

### Web Search Node Features
- ‚úÖ Two modes:
  - **Search Mode**: Query text ‚Üí Returns search results array
  - **URL Mode**: Query is URL ‚Üí Fetches and scrapes content from URL
- ‚úÖ Max results configuration (1-20)
- ‚úÖ Returns structured array with:
  - `title` - Page title
  - `url` - Source URL
  - `content` - Full extracted text content
  - `snippet` - Preview (first 200 chars)
  - `metadata` - Additional information
- ‚úÖ Works seamlessly with Loop node for batch processing
